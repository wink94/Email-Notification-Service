get /v1/notification/email/push-notification
Sends a Notification to the Recipient (1)

Operation Behavior.

This operation allows users to send email notifications to recipients.

Inputs

The operation expects the

following parameters:

  - `sender` (required): The email address of the sender. 
  - `recipients` (required): An array of email addresses of the recipients. 
  - `subject` (required): The subject line of the email. 
  - `message` (required): The content of the email message. 
  - `attachments` (optional): An array of file attachments to include in the email.

Outputs

The operation does not return any specific output. A successful request will result in the email notification being sent to the recipients' email addresses. ### Behavior

  1. The operation validates the sender parameter to ensure it is a valid email address.
  2. The operation validates the recipients parameter to ensure it contains at least one valid email address.
  3. The operation validates that the subject and message parameters are not empty.
  4. If the validation passes, the operation composes the email with the provided information.
  5. If attachments are specified, they are attached to the email.
  6. The operation sends the email using the configured email service provider or SMTP server.
  7. If the email is successfully sent, a response with a 200 status code is returned.
  8. If any errors occur during the email sending process, an appropriate error response with a corresponding status code is returned.

  1. The operation validates the sender parameter to ensure it is a valid email address.
  2. The operation validates the recipients parameter to ensure it contains at least one valid email address.
  3. The operation validates that the subject and message parameters are not empty.
  4. If the validation passes, the operation composes the email with the provided information.
  5. If attachments are specified, they are attached to the email.
  6. The operation sends the email using the configured email service provider or SMTP server.
  7. If the email is successfully sent, a response with a 200 status code is returned.
  8. If any errors occur during the email sending process, an appropriate error response with a corresponding status code is returned.
